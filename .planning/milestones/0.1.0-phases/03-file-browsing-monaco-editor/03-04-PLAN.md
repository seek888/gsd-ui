  <name>Task 4: Handle unsaved changes prompt on file switch</name>
  <files>src/components/documents/FileTree.tsx</files>
  <action>
    Update src/components/documents/FileTree.tsx to prompt user when switching away from a dirty file.

    This is a "nice to have" safety feature. Implement a simple version:

    In the file click handler (handleClick):
    - Check if current openFile exists and is dirty
    - If dirty, show browser confirm dialog before opening new file
    - If user cancels, don't open new file

    Implementation:
    ```tsx
    const handleClick = (node: Node<FileNode>) => {
      if (node.data.type === 'folder') {
        node.toggle();
        return;
      }

      const { openFile, openFile: openFileAction } = useFileStore.getState();

      if (openFile?.isDirty && openFile.path !== node.data.path) {
        const proceed = confirm('You have unsaved changes. Do you want to continue?');
        if (!proceed) return;
      }

      openFileAction(node.data.path);
    };
    ```

    This is a basic implementation. A more sophisticated version would use a custom dialog component (can be added later).

    Note: The confirm dialog is browser-native, which works in Tauri. For a more native feel, consider using Tauri's dialog API in a future enhancement.

    If this prompt feels too intrusive, it can be made optional or removed. The key requirement (FILE-04) is the visual indicator and save button state, which are already implemented.
  </action>
