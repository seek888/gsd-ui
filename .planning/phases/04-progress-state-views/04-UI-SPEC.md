## Component Specifications

### PhaseCard Component
**Purpose:** Display individual phase with progress and status

**Layout:**
- Card container with rounded-lg border, shadow-sm
- Header: Phase name (text-lg font-semibold) + Status badge (top-right)
- Progress section: Progress bar + percentage text (e.g., "79%")
- Footer: Plan count (e.g., "3/5 plans complete")

**States:**
- Default (collapsed): Shows phase info only
- Expanded: Shows list of plans below
- Highlighted: Active phase gets subtle border accent (ring-2 ring-ring)

**Interaction:** Click anywhere on card to expand/collapse

**Source:** D-01 to D-07 decisions

### PlanItem Component
**Purpose:** Display individual plan within expanded phase

**Layout:**
- Row with plan ID (e.g., "04-01"), plan name
- Completion indicators: PLAN.md (checkmark icon), SUMMARY.md (checkmark icon)
- Status text: "未开始" / "进行中" / "已完成"

**States:**
- Not started: Gray icons, no checkmarks
- In progress: One or both files exist, mixed icons
- Complete: Both files exist, green checkmarks

**Source:** D-10 decision

### SessionContext Component
**Purpose:** Display current session state from STATE.md

**Layout:**
- Separate section below phase cards (scrollable)
- Distinct background (bg-muted/50 or border-top)
- Three subsections: Recent Position, Blockers, Key Decisions
- Each subsection has icon + heading + content

**Content Display:**
- Recent Position: "阶段 {phase} - {activity}"
- Blockers: List or "无阻塞项" message
- Key Decisions: Summary count or recent decision excerpts

**Source:** D-11 to D-14, D-16 decisions

### AttentionPanel Component
**Purpose:** Aggregate items needing user attention

**Layout:**
- Can be embedded in SessionContext or separate card
- List of items with icons:
  - Warning icon: Incomplete plans
  - Alert icon: Blockers from STATE.md
  - Info icon: Outstanding items
- Each item is clickable (links to relevant plan/phase)

**Source:** D-18, D-19 decisions

## Interaction Patterns

### Accordion Behavior (DECISION)
**Choice:** Multi-expand (multiple phases can be open simultaneously)

**Rationale:** Allows comparing plan details across phases without re-expanding. More flexible for power users.

**Animation:** CSS transition on max-height (300ms ease-in-out)

**Source:** Claude's discretion (D-09)

### Progress Visualization
- Horizontal progress bar (div with width %)
- Smooth transition on width changes (transition-all duration-300)
- Percentage text right-aligned to progress bar
- Color changes based on completion: gray (0%) -> yellow (1-99%) -> green (100%)

**Source:** D-04 to D-07 decisions

### Data Refresh
- Progress data fetched on view mount (DashboardView initial render)
- Manual refresh via button (optional, for future enhancement)
- Loading state shows skeleton cards or spinner
- Error state shows error message with retry option

**Source:** D-15 to D-17 decisions

## Accessibility

- All cards use button element or have role="button" with tabIndex
- Expanded/collapsed state communicated via aria-expanded
- Status badges use aria-label for screen readers
- Progress bar uses role="progressbar" with aria-valuenow
- Keyboard navigation: Tab through cards, Enter/Space to expand

**Source:** Radix UI primitives (Accordion, ScrollArea) provide accessibility foundation
