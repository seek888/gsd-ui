<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (webview) -> Tauri backend | Untrusted web content crosses here via IPC |
| Tauri backend -> OS | Tauri backend executes shell commands and file operations on behalf of the webview |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Tauri capabilities | mitigate | Capabilities are scoped to specific plugins. Shell plugin allows `claude` CLI only (via scope). No arbitrary command execution. |
| T-01-02 | Information Disclosure | FS plugin scope | mitigate | FS plugin scoped to user-selected project directory only. No broad file system access granted. |
| T-01-03 | Denial of Service | Shell plugin | accept | User can run any GSD command they choose. This is the intended behavior. |
| T-01-04 | Elevation of Privilege | CSP configuration | mitigate | CSP has `unsafe-eval` for Monaco Editor. Monaco is only loaded when editing files, not by default. Monaco CSP risk is accepted for functionality. |
</threat_model>
