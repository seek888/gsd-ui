## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Markdown file content → react-markdown | Untrusted Markdown from user's project crosses into HTML rendering |
| Code block content → SyntaxHighlighter | Code snippets may contain malicious patterns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | Markdown rendering | accept | Low risk - displaying user's own content back to them |
| T-03-08 | Tampering | XSS via malicious Markdown | mitigate | react-markdown escapes HTML by default (do NOT set skipHtml: true). Only render <code> and <pre> with trusted content |
| T-03-09 | Repudiation | N/A | accept | Preview is read-only, no modifications in this plan |
| T-03-10 | Information Disclosure | N/A | accept | User views their own files |
| T-03-11 | Denial of Service | Malicious Markdown causing re-render loops | mitigate | ReactMarkdown has built-in safeguards. Limit content size if needed (truncate >1MB files) |
| T-03-12 | Escalation | N/A | accept | No privilege escalation in read-only preview |

</threat_model>
