<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → File System | User-selected project path crosses into file system read operations |
| File tree → Tauri fs plugin | File paths from directory scanning cross into Tauri's Rust fs layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | File tree node rendering | accept | Low risk - only displaying file names, no authentication involved |
| T-03-02 | Tampering | File path construction (readDir) | mitigate | Use Tauri's fs.join() to construct paths, never concatenate user input directly. fs.readDir() validates paths against capability ACL |
| T-03-03 | Repudiation | N/A | accept | File browsing is read-only in this plan, no audit trail needed |
| T-03-04 | Information Disclosure | File tree contents | mitigate | Tauri capability system restricts access to .planning/ only. Files outside this scope cannot be read |
| T-03-05 | Denial of Service | File tree with thousands of files | mitigate | react-arborist provides virtualization (renders only visible nodes). Limit initial directory scan depth if needed |
| T-03-06 | Escalation | N/A | accept | No privilege escalation in read-only file browsing |

</threat_model>
