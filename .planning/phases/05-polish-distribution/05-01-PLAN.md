创建 `src/lib/errorHandler.ts` 实现分级错误处理策略：

1. **错误分类** (D-01):
   - `ErrorCategory.RETRYABLE`: 网络相关、进程相关错误（自动重试）
   - `ErrorCategory.CRITICAL`: CLI 未安装、文件权限拒绝（立即通知）

2. **重试逻辑** (D-04):
   - `retryable<T>(fn: () => Promise<T>, category: ErrorCategory): Promise<T>`
   - 最多重试 3 次
   - 间隔递增: 1000ms → 2000ms → 3000
   - 仅对 RETRYABLE 类别进行自动重试

3. **错误处理函数**:
   - `handleError(error: Error, category: ErrorCategory, context?: string): void`
   - 根据类别决定是否重试、如何展示

创建 `src/lib/logger.ts` 实现双日志记录 (D-03):

1. **文件日志配置**:
   - 日志目录: 用户目录下的 `.gsd-ui/logs/`
   - 文件命名: `error-{YYYY-MM-DD}.log`（每天一个文件）
   - 使用 Tauri fs plugin 写入文件

2. **日志函数**:
   - `logError(error: Error, context?: string): void`
   - `logToFile(message: string): Promise<void>`
   - 同时输出到 console.error 和文件日志

3. **日志格式**:
   ```
   [YYYY-MM-DD HH:mm:ss] [ERROR] {context}
   {error.message}
   {error.stack}
   ```

**依赖**: 新增 `@tauri-apps/plugin-fs` 用于日志文件写入（已在项目中）

创建 `src/stores/errorStore.ts` 管理应用级别的错误状态：

**State 接口**:
```typescript
interface ErrorItem {
  id: string;
  category: ErrorCategory;
  message: string;
  context?: string;
  timestamp: number;
  displayType: 'modal' | 'toast' | 'inline';  // D-02
}

interface ErrorState {
  errors: ErrorItem[];
  addError: (error: Error, category: ErrorCategory, context?: string) => void;
  clearError: (id: string) => void;
  clearErrors: () => void;
  getErrorsByDisplayType: (type: 'modal' | 'toast' | 'inline') => ErrorItem[];
}
```

**关键行为**:
- `addError`: 自动生成 UUID 作为 id，根据 category 映射到 displayType
  - CRITICAL → modal
  - RETRYABLE → toast
  - 其他 → inline
- `getErrorsByDisplayType`: 过滤返回指定展示类型的错误
- 使用 `create<ErrorState>((set, get) => ({ ... }))` 模式

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → App | 外部命令输出可能包含恶意内容 |
| File System → App | 用户选择的目录可能包含非预期文件 |
| Log Files → App | 日志文件写入可能失败或被篡改 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | S (Spoofing) | CLI output | mitigate | 所有 CLI 输出都作为纯文本处理，不作为代码执行 |
| T-05-02 | T (Tampering) | Log files | accept | 日志仅用于诊断，不影响应用逻辑 |
| T-05-03 | R (Repudiation) | Error logging | mitigate | 错误日志包含时间戳和上下文，便于审计 |
| T-05-04 | I (Information Disclosure) | Error messages | mitigate | 避免在错误消息中泄露敏感路径或系统信息 |
| T-05-05 | D (Denial of Service) | Retry logic | mitigate | 重试次数上限为 3 次，防止无限循环 |
| T-05-06 | E (Elevation of Privilege) | File operations | accept | Tauri 沙箱已限制文件系统访问范围 |
</threat_model>
