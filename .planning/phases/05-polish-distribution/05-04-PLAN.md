验证和测试 Windows 路径处理：

**检查现有代码的跨平台兼容性**:
1. `src/lib/shell.ts` 中的 PATH 扩展:
   - 当前使用 `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` (macOS/Linux)
   - 需要为 Windows 添加不同的 PATH 逻辑

2. Windows 路径处理:
   - Windows 使用反斜杠 `\` 作为路径分隔符
   - Tauri 的 path API 应该处理跨平台路径
   - 验证 `join` 和其他路径操作在 Windows 上正常工作

**需要的修改**:
在 `src/lib/shell.ts` 中添加平台检测:
```typescript
import { platform } from '@tauri-apps/plugin-os';  // 或使用其他方式检测

// 在 checkClaudeInstalled 中:
const isWindows = platform() === 'windows';
const searchPaths = isWindows
  ? [
      // Windows CLI 路径（npm 全局安装位置）
      'C:\\Users\\%USERNAME%\\AppData\\Roaming\\npm',
      'C:\\Program Files\\nodejs',
    ]
  : [
      '/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin',
      '/usr/local/bin:/usr/bin:/bin',
    ];
```

**注意**: 如果 `@tauri-apps/plugin-os` 不可用，可以使用其他方式检测平台。

如果当前代码没有平台检测，添加基本的 Windows 支持。
