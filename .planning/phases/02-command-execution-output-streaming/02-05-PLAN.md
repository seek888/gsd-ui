3. Add scroll handler using xterm.js onScroll event (per RESEARCH.md Pattern 3):
   ```typescript
   useEffect(() => {
     if (!terminalInstanceRef.current) return;

     const handleScroll = () => {
       const term = terminalInstanceRef.current;
       if (!term) return;

       // Get buffer and viewport position
       const buffer = term.buffer.active;
       const viewportY = term._core._viewport?._ydisp || 0;
       const bufferHeight = buffer.length;
       const threshold = 100; // Per D-12

       const isNearBottom = bufferHeight - viewportY < threshold;

       setUserScrolled(!isNearBottom);
     };

     term.onScroll(handleScroll);

     return () => {
       term.onScroll(() => {}); // Cleanup
     };
   }, []);
   ```

4. Add auto-scroll on new data:
   - Create a ref to track if user has scrolled
   - In cliStore writeStdout/writeStderr, check userScrolled state
   - Only auto-scroll if !userScrolled

   However, since cliStore is separate from component, use a different approach:
   - Create a callback prop onScrollStateChanged in TerminalOutput
   - Pass to cliStore or use window event

   Better approach per RESEARCH.md Pattern 3:
   - Track scroll state in component only
   - Use terminal.writeln() which auto-scrolls by default
   - Disable auto-scroll by using terminal.write() without newline when userScrolled=true

5. Simplified implementation:
   - Store scroll state in component, not store
   - Always let xterm.js handle auto-scroll (default behavior)
   - Show JumpToBottom when userScrolled=true
   - JumpToBottom.onClick calls terminal.scrollToBottom() and resets userScrolled

6. Handle scroll-to-bottom callback:
   ```typescript
   const scrollToBottom = () => {
     terminalInstanceRef.current?.scrollToBottom();
     setUserScrolled(false);
   };
   ```

   return (
     <div className="relative h-full w-full">
       <div ref={terminalRef} className="h-full w-full" style={{ minHeight: '100%' }} />
       <JumpToBottom visible={userScrolled} onClick={scrollToBottom} />
     </div>
   );
   ```

- Store last write position, compare to buffer.length
- Or use terminal.onData (user input) vs terminal.onWrite (programmatic)
- Or use Terminal's registerDecoration API for scroll position

Fallback if internal API unavailable:

   const shouldAutoScrollRef = useRef(true);
   ```

2. Update handleScroll to set the ref:
   ```typescript
   const handleScroll = () => {
     const term = terminalInstanceRef.current;
     if (!term) return;

     const buffer = term.buffer.active;
     const viewportY = term._core._viewport?._ydisp || 0;
     const bufferHeight = buffer.length;
     const threshold = 100;

     const isNearBottom = bufferHeight - viewportY < threshold;
     shouldAutoScrollRef.current = isNearBottom;
     setUserScrolled(!isNearBottom);
   };
   ```

3. Pass shouldAutoScroll callback to parent or store:
   - Since cliStore writeStdout/writeStderr don't have access to component state,
     we need to either:
     a) Create a callback prop onTerminalReady that returns an object with shouldAutoScroll getter
     b) Use a global ref or event system
     c) Accept that auto-scroll always happens (xterm.js default) and we just show the button

   Approach (c) is simplest and matches RESEARCH.md: xterm.js auto-scrolls by default on write().
   The "pause" is visual feedback (showing the button), not actually suppressing writes.

   For actual suppression, we'd need to intercept writes, which is complex.
   The MVP interpretation per D-12: "pause on user scroll up" means we show the button,
   not that we prevent new output from appearing.
